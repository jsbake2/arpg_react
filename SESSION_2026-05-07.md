## What this tool is now

Single-user-per-account companion for ARPGs that does three things:

1. **Event timers** (D4 only) — Helltide / Legion / Realmwalker / World
   Boss countdowns + audio + notify. Already worked, kept intact.
2. **Rule-engine auto-cast** — per-build JSON rules; the daemon
   screen-grabs the skill bar at 4Hz, classifies each slot
   (READY/ACTIVE/COOLDOWN), evaluates rules top-down, presses keys.
   Game-aware: D4 detector is tuned, POE2 calibration in progress.
3. **OCR-driven skill timings** — point-and-shoot calibration tool that
   reads each skill's Use Time / Cooldown / Duration from the in-game
   panel, writes them to the build, daemon respects them as throttle
   gates so we don't hammer the game with presses the engine "thinks"
   landed but actually got eaten by the GCD.

Hosted at **`https://arpg.jsb-emr.us`** (formerly `d4.jsb-emr.us`, kept
as alias). Two seeded users: `jbaker` and `matt`, both `arpg123`.

## Architecture pieces

### Daemon (`arpg_react/daemon.py`)
- Main loop at 250ms tick.
- Single `Detector.detect()` per tick — one `grim`-piped PPM screen
  grab (~25ms vs PIL.ImageGrab's 570ms PNG-via-tempfile dance),
  patches sampled in-memory via `img.load()`.
- Rule engine consumes detector readings; pushes `slot_states`,
  `resource_fills`, `boss_detected` into `EvalContext`.
- Background `editor-sync` thread polls the editor backend every 60s
  (when `D4_EDITOR_PASSWORD` env or `editor.password` file is set).
- IPC streams `status` + `alert` + `debug` frames to connected panels.
- Game-state gates input: `MENU`, `MOUNTED`, `TOWN` all halt auto-cast
  even when WATCHER is on. AUTO/ON/OFF override still wins.

### Detector (`arpg_react/watchers/detector.py`)
- All thresholds + slot coords in `DetectorConfig` (currently D4-only).
- `GameState`: `combat | town | mounted | menu | unknown`.
- `SlotStatus`: `ready | active | cooldown | unknown`.
- Three slot states mapped to engine's `SlotState` enum.
- Boss + mount detection via patch-match templates encoded in
  `detector_refs.py` (BOSS_REF, MOUNT_REF). HP-independent — the
  templates sit on UI frames, not on the HP fill, so they keep
  matching as the boss takes damage.
- HP/mana orbs via vertical-column saturation count, no per-build
  config needed (works regardless of class palette).
- TOWN gate: max body saturation across all slots < 0.15 — every icon
  rendered as stone-grey.

### Web editor backend (`/tmp/d4-app/app.py` deployed to
`/home/jbaker/d4-rule-editor-app/` on 10.0.0.16)
- FastAPI + SQLite. Single source of truth for builds.
- Schema: `users`, `sessions`, `builds(owner, game, name, data)`,
  `keymaps(owner, game)`. Old single-PK `builds` table auto-migrated
  to `(jbaker, d4)` on first start.
- Auth: session cookie for the web UI (POST `/api/login`); HTTP Basic
  for the daemon (back-compat with the legacy `d4123d4` shared password
  — maps to `jbaker` + `d4`).
- `Cache-Control: no-cache` for `*.js/*.css/*.html` — stops browsers
  holding stale assets through deploys.
- Server-side condition sanitizer on PUT (defense vs editor JS sending
  garbage `target=null, value=0.5` for `SLOT_STATE_IS` conditions).

### Web editor frontend (`/tmp/d4-app/static/`)
- Login page → game picker (D4 / POE2 with stylized SVG tiles) →
  rule editor.
- Rule editor tabs: `BUILD`, `SKILLS`, `RULES`, `POTION`. **No
  CALIBRATION or RESOURCES tabs** — those were removed because all
  detector coords are auto-calibrated.
- `SKILLS` tab: per-slot rows with `cast_ms / recast_ms / active_ms`
  inputs. Defaults to all-zero (instant) so existing builds need no
  changes.
- Per-game `HOTKEYS` set: D4 = `1,2,3,4,L,R`, POE2 = `LMB,MMB,RMB,Q,E,
  R,T,F`. Determines what shows in SKILLS tab + RULES dropdowns.

### Panel app (`arpg_react/panel/`)
- Three tabs: `POE2`, `D4`, `TIMERS`. Default is D4 (the only
  fully-wired game).
- POE2 tab is a placeholder pointing at `arpg_stuff/poe2_calibration_<user>.png`.
- D4 tab = build picker + class banner + debug console (live log
  stream from the daemon, `INFO+` records via `_IPCLogHandler`).
- TIMERS tab = the four event cards (always D4-flavoured; available
  regardless of the active game tab).

### Calibrator (`arpg_react/calibrator.py`)
- Standalone Qt window: `arpg-react calibrate-skills`. No CLI build
  arg required.
- Top bar: connection info (`arpg.jsb-emr.us · jbaker`), GAME
  dropdown, BUILD dropdown (lists builds from server), SYNC button.
- Per-slot rows with CAPTURE buttons + 3 spinboxes (cast/recast/active).
- SAVE button: `PUT /api/builds/<name>?game=<g>` — writes timings to
  the editor database directly. **No local file involvement.**
- API-driven: GET on build switch, PUT on save.
- Default game: `poe2`. Slot set rebuilds when game changes.

## OCR — the messy part

POE2's stylized panel font confuses Tesseract in characteristic ways.
We catch the common cases via a small heuristic in
`arpg_react/skill_ocr.py::_smart_parse`. Calibration rig:

- `_grab_screen(bbox=...)` — `grim -g <rect>` for region-only capture.
  Way more reliable than full-screen because the calibrator window /
  desktop / chat noise no longer feed into the OCR.
- `_preprocess_for_ocr` — grayscale → threshold @ 130 → 2x upscale.
  Cleans the stylized POE2 font.
- Tesseract config: `--psm 6` (assume uniform block of text).
- POE2 default OCR bbox (jbaker, 2560×1440): `(914, 311, 1641, 1036)`
  — the right-side skill detail panel.
- Override at runtime via `D4_OCR_BBOX="x1,y1,x2,y2"` env var.

### Regex patterns

```
PAT_CAST   = (?:cast|use)\s*time\s*[:\-]?\s*<num><tail>
PAT_RECAST = cool\s*down(?:\s*time)?\s*[:\-]?\s*<num><tail>
PAT_ACTIVE = (?:[A-Za-z]+\s+){0,4}duration\s*[:\-]?\s+<num><tail>
```

Where `<num> = (\d+(?:[.,]\d+)?)` and `<tail> = \s*(?:sec(?:onds?)?|s)?`.
POE2 uses "USE TIME" not "Cast Time", so PAT_CAST accepts both. POE2
prefixes "Duration" with a per-skill word (`SKILL EFFECT DURATION`,
`AURA DURATION`, `WALL DURATION`, `SIGIL DURATION`, `CURSE DURATION`,
`ORB DURATION`, `EXPOSURE DURATION` — all observed). `PAT_ACTIVE`
takes 0-4 leading words to swallow these.

### Known OCR misreads (Tesseract on POE2 font)

| panel text | Tesseract returns | heuristic action |
|---|---|---|
| `Cooldown 10s` | `Cooldown 108` or `Cooldown 108\nS` | strip trailing `8` (len ≥ 3) → 10000ms |
| `19.23s` | `19.235` | strip trailing 5/8 from 3+ decimal places → 19230ms |
| `Cast Time 0.53s` | `Cast Time 0.538` | same decimal-strip → 530ms |
| `Cooldown 0s` | `Cooldown 0s` | matches cleanly, no correction |
| `Use Time 1.5s` | `Use Time 15` | **NOT** auto-corrected — short integer ending in 5 stays. User hand-edits. |

`_smart_parse` rules:
- **Integer**: strip trailing `8` if length ≥ 3 (so `8`, `18`, `28`,
  `58` stay as plausible legit cooldowns).
- **Decimal**: strip trailing `5` or `8` if it's the THIRD or later
  fractional digit (POE2 always uses 2 decimal places — over-precise
  reads are misreads).
- Tradeoff: a real 108s cooldown gets stripped to 10s. User hand-edits
  if they spot it.

### Always-on OCR debug dump

Every capture writes the full Tesseract output to
`/tmp/arpg_react_ocr_<unix_ts>.txt`. The matched substrings are also
logged at INFO. Keep this on while we shake out more misread patterns.
