Identity

Skill-bar slot positions, orb fill levels, and boss / mount detection are auto-calibrated against the in-game UI — no per-build pixel coordinates or colors are needed here. Just rules.

Skills (per-slot timing)

All defaults are 0 (instant) — leave them alone for skills with no cast or cooldown. Set non-zero values when you want the engine to wait the right amount of time:
cast — animation time after the press; combo steps queued after this slot wait at least this long.
recast — minimum gap before the same slot can fire again (per-skill GCD).
active — buff duration; informational for now.

Rules (top-down precedence)

Each tick, rules are evaluated top-to-bottom. First rule whose conditions are met fires; rules below it skip. Drag the ☰ handle to reorder.

Potion

Hotkey + auto-trigger threshold. The daemon presses this key when health drops below the configured percentage (subject to potion cooldown).

Profile (per-user, per-game environment)

Settings here are not tied to any one build — they describe your machine + key bindings for this game and are reused across every build. The daemon translates rule slot names through the keymap before pressing keys, and scales detector + OCR coordinates by your resolution and UI scale.

Display

Match the values to your actual game window. UI scale is the in-game slider (D4 → Options → Gameplay → Interface Size; POE2 → Options → UI → Scale).

Keymap

Slot names are the labels rules and skills are written against. The actual key is what the daemon presses when a rule fires for that slot. Defaults are identity (e.g. Q → q); if you bound your skill to 1 in-game instead of Q, change the right column to 1.
